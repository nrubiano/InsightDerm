1. Overview To Top

Rating is a very important both for us (it motivates to develop new modules), and for future customers. So we request you to rate our Templates with 5 Stars on Themeforest. Thank you!

Clean UI is the first framework in the world, which uses the concept of modularity. The use of such approach allows you to drastically reduce the time and cost of development. Build the interface from the ready blocks and modules quickly and easily, like Lego! And all this is available now for Angular (version 2 or higher).

This is a completely modular framework, built on the latest technologies, such as BEM methodology (Block Element Modifier). You can learn more about the methodology here. Following the simple and understandable rules of this methodology allows the easy support of code even after long-term development. Adding new functionality and modifying the old one has never been so easy before. The presence of a variety of useful development methodologies make this process easier and more enjoyable; for example, such feature as rem measurement units make mobile versions of applications, built on the basis of Clean UI, look great on any type of devices. Open our template on your mobile phone and be sure to verify it yourself!

Clean UI - a modern professional admin template, based on Bootstrap 4 framework. It's a powerful and super flexible tool, which suits best for any kind of web application: Web Applications; CRM; CMS; Admin Panels; Dashboards; etc. Clean UI is fully responsive, which means that it looks perfect on mobiles and tablets.

Our template is fully based on SASS pre-processor, includes 50+ commented SASS files. Each file corresponds to a single component, layout, page, plugin or extension – so you can easily find necessary piece of code and edit it for your needs. The package includes CSS files compiled from SASS.

3. HTML Version To Top

For use px instead rem in CSS, change calculation formula in rem function (version_html/src/modules/core/css/mixins.scss) and recompile SASS by gulp build:css task.

Use the ready (pre-compiled) templates of different versions from the folder version_html/dist/versions/

or

Use Gulp for build source files:

  1. Install latest Node.js from here https://nodejs.org;
  2. Install Bower npm install --global bower gulp;
  3. Install Node modules npm install;
  4. Install project dependencies bower install;
  5. Run gulp gulp (cборка происходит в папку version_html/dist/);
  6. If a browser did not open automatically, please open http://localhost:9000 in any browser.

Gulp tasks:

  1. gulp default task, includes ['build', 'watch', 'serve'] задачи;
  2. gulp build includes ['build:structure', 'build:vendors', 'build:js', 'build:css', 'build:img', 'build:fonts', 'build:templates', 'build:txt'];
  3. gulp serve project serve;
  4. gulp build:structure build pages and structure (html);
  5. gulp build:vendors copy third-party plugins;
  6. gulp build:js build js scripts;
  7. gulp build:css compilation and build sass files (css);
  8. gulp build:img build images;
  9. gulp build:fonts build fonts;
  10. gulp build:templates build templates from the modules;
  11. gulp build:txt build txt files.



4. Angular Version To Top

Template version adapted for Angular (version 2 and higher). Application routing + Ng-Bootstrap directives. Use of this version is optional.

Installation:

  1. Install latest Node.js from here https://nodejs.org;
  2. Install Angualr CLI npm install -g angular/cli;
  3. Enter Angular directory cd version_angular/;
  4. Install Node modules npm install;
  5. Run Angular ng serve;
  6. Build Angular production files ng build --prod --aot (cборка происходит в папку version_angular/dist/);
  7. Open in any browser http://localhost:4200



5. Template Structure To Top

The original display of template consists of the following blocks:

  1. Top-bar - block, based on the "Top-bar Module";
  2. Left Sidebar - block, based on the "Menu-left Module";
  3. Content - content block;
  4. Right Sidebar - block, based on the "Menu-right Module";
  5. Footer - block, based on the "Footer Module".

Code structure:

  • Each class has the "cat__" prefix (abbreviation of Cleanui Admin Template); this makes any class unique, which allows to easily add modifiers, as well as get rid of unwanted styles inheritance;
  • Please see the section "Template configuration" for the information on the possible classes of body tag;
  • The template can contain any set of modules and in any order.

<html>
<head>
    <!-- meta, scripts, and styles includes here -->
</head>
<body class="">
    <div class="cat__menu-right">
        <!-- "menu-right module" template -->
    </div>
    <div class="cat__menu-left">
        <!-- "menu-left module" template -->
    </div>
    <div class="cat__top-bar">
        <!-- "top-bar module" template -->
    </div>
    <div class="cat__content">
        <!-- current page content -->
        <!-- current page scripts -->

        <div class="cat__footer">
            <!-- "footer module" template -->
        </div>
    </div>
</body>
</html>



6. Template Configuration To Top

The template has a set of global settings, which are activated by adding certain classes to the bodytag:

  • These settings may affect the modules connected to the template. Information on any impact of a particular global setting on the module is located in the description of this module;
  • These settings can be used for any of the modules and modify them regardless of the names of these settings.

Settings:

Class Applies to By default used for
.cat__config--vertical body The vertical layout type. Mandatory class, can be replaceed by .cat__config--horizontal
.cat__config--horizontal body The horizontal layout type. Mandatory class, can be replaceed by .cat__config--vertical
.cat__config--boxed body Fixed width content block
.cat__config--compact body Compact layout type
.cat__config--unfixed body Layout type with unfixed blocks
.cat__config--superclean body Clean styles layout type
.cat__config--squared-corners body Squared corners
.cat__config--shadow body Drop shadows

Example of use (will configure the compact horizontal menu on top with the fixed width of content-area):

<body class="cat__config--horizontal cat__config--boxed cat__config--compact">

.cat__config--unfixed body Activates unfixed blocks layout type for "Core Module", "Menu-left Module", "Menu-left Module"

Clean UI - это первый в мире фреймворк, который использует концепцию модульности. Использование такого подхода позволяет резко сократить сроки и стоимость разработки. Стройте интерфейс из готовых блоков и модулей быстро и легко, подобно конструктору Lego! И все это доступно теперь и для Angular (версии 2 или выше).

Это полностью модульный фреймворк, построенный на базе последних технологий, таких как методология BEM (блок-элемент-модификатор). Подробнее об этой методологии можно прочитать здесь. Следование простым и понятным правилам этой методологии позволяет без проблем поддерживать код даже после длительной разработки. Добавление новой функциональности и изменение старой никогда не было таким простым прежде. Присутствие разнообразных полезных методик разработки делают этот процесс легким и приятным, например, такая особенность, как единицы измерения rem делают мобильные версии приложений построенных на базе Clean UI отлично выглядящими на любом типе устройств. Откройте наш шаблон в своем мобильном телефоне и убедитесь в этом сами!

Clean UI - современный профессиональный шаблон, построенный на базе фреймворка Bootrap 4. Это очень функциональный и гибкий интсрумент который подойдет для любого типа веб-интерфейса: Веб-приложение; CRM; CMS; Панели администратора, Дашбоарды и т.д. Clean UI полностью адаптирутея под любой размер экрана, что означает удобное использование будь то на мобильно телефоне или планшете.

Наш шаблон полностью сделал на базе SASS препроцессора стилей, включает в себя более 50 SASS файлов с комментариями к коду. Каждый файл относится к определенному компоненту, шаблону, странице или расширению - это значит что вы можете легко найти и редактировать любую часть кода. Также в нашем шаблоне присутсвуют CSS файлы, заранее скомпилированные из SASS файлов.

ВНИМАНИЕ: Каждая Стандартная лицензия может быть использована только на для одного домена или для одного клиента. Мы надеемся вы будете следовать этим правилам, что гарантирует вам длительную подержки и бесплатные обновления шаблона. Так же не забывайте о существовании Расширенной лицензии.

Для сборки исходного кода используйте Gulp:

  1. Установить последнюю версию Node.js отсюда https://nodejs.org;
  2. Установить Bower npm install --global bower gulp;
  3. Установить необходимые модули Node npm install;
  4. Установить зависимости проекта bower install;
  5. Запуск gulp gulp (cборка происходит в папку version_html/dist/);

Список Gulp задач:

  1. gulp задача по умолчанию, включает в себя ['build', 'watch', 'serve'] задачи;
  2. gulp build включает в себя ['build:structure', 'build:vendors', 'build:js', 'build:css', 'build:img', 'build:fonts', 'build:templates', 'build:txt'];
  3. gulp serve запуск live-разработки;
  4. gulp build:structure сборка шаблонов и структуры проекта (html);
  5. gulp build:vendors объединение и копирование сторонних плагинов;
  6. gulp build:js сборка js скриптов (javascript);
  7. gulp build:css компиляция и сборка sass файлов (css);
  8. gulp build:img сборка картинок (images);
  9. gulp build:fonts сборка шрифтов (fonts);
  10. gulp build:templates копирование шаблонов из модулей;
  11. gulp build:txt копирование текстовых файлов из модулей.

6. Конфигурация шаблона Наверх

В шаблоне присутствует набор глобальных настроек, которые активируются добавлением определенных классов к тегу body:

  • Эти настройки могут влиять на подключенные в шаблон модули. Информацию о каком-либо влиянии глобальной настройки на модуль читайте в описании этого модуля;
  • Эти настройки могут использоваться для любых модулей и изменять их вне зависимости от названия этих настроек.

Настройки:

Класс Применяется к Назначение по умолчанию
.cat__config--vertical body Вертикальный тип отображения. Обязательный класс, может быть заменен на .cat__config--horizontal
.cat__config--horizontal body Горизонтальный тип отображения. Обязательный класс, может быть заменен на .cat__config--vertical
.cat__config--boxed body Фиксириванная ширина контент-области
.cat__config--compact body Компактный тип отображения
.cat__config--unfixed body Открепленные блоки
.cat__config--superclean body Облегченный тип отображения
.cat__config--squared-corners body Квадратные углы
.cat__config--shadow body Отбрасывание теней

Пример использования (сконфигурирует находящееся сверху компактное горизонтальное меню с фиксированной шириной контент-области):

<body class="cat__config--horizontal cat__config--boxed cat__config--compact">
